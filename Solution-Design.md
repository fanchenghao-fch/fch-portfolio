## 0. 文档说明与范围

本文档承接已确认的《需求分析文档》与已交付的 UI 原型，定义**技术实现层**的方案：技术选型（每项均经联网调研、给出注解与推荐）、系统架构、各模块对照原型的实现路径、数据模型、API、安全防护、部署与备案、成本预算，以及开发环境准备（MCP / Skills）。

三条硬约束贯穿全文：

1. **UI 完全以提供的原型为准，不自行发挥。** 选型与实现都优先保证原型 HTML/CSS 能 1:1 落地。
2. **LLM API key 由本人持有、仅存服务端**，前端永不接触；所有 LLM 调用经服务端代理并限流。
3. **先海外快速上线，国内备案并行**（需求文档已确认）。

> 调研时间为 2026 年 6 月，价格/版本以各官方页面实时为准；本文所列为调研当时数据，落地前请复核。

## 1. 总体架构

```
                 访客（HR / 用人方）

        │  02-home / 03-detail（原型 1:1）   │
        └───────┬───────────────┬──────────┘
                │               │

   │  · 持有 LLM key  · 限流封顶  · 鉴权          │
   └───────┬───────────────┬──────────┬─────────┘
           │               │          │
     ┌─────▼────┐   ┌──────▼─────┐  ┌─▼──────┐

     └──────────┘   └──────┬─────┘  └────────┘
```

核心思想：**展示层尽量静态（快、稳、好备案）**，**动态能力收敛到少量服务端 API**，LLM、数据库、邮件均为后端依赖，前端零密钥。

## 2. 技术选型详解（逐项调研 + 推荐）

下表为最终推荐栈，逐层论证见后。

| 层 | 推荐方案 | 一句话理由 |
|---|---|---|
| 前端框架 | **Astro** | 现有原型 HTML/CSS 可近乎原样落地，默认零 JS，满足"UI 1:1" |
| 服务端 | **Astro API Endpoints**（Vercel Functions 承载） | 与前端同仓，少量接口无需独立后端 |
| 数据库 | **Supabase** | Postgres + 内置 Auth（后台登录）+ 免费额度足够 |
| LLM | **智谱 GLM-4.7-Flash**（免费）；付费备选 DeepSeek V4-Flash | 低流量个人站免费即可，国内直连 |
| RAG | **初期全量语料注入**；语料增长后 pgvector + 硅基流动 bge-m3 | 语料仅 5–8 项目，无需重型向量库 |
| 邮件 | **Resend** | 免费额度足、API 简洁；自收信亦可用 SMTP |
| 部署一期 | **Vercel** + Cloudflare/EdgeOne 优选 IP | 当天上线、零备案 |
| 部署二期 | **腾讯云 EdgeOne Pages** 或国内轻量云 + **ICP 备案** | 国内访问快而稳 |
| 鉴权 | **Supabase Auth**（后台）/ 一次性表单（门禁） | 后台安全，门禁体验轻 |

### 2.1 前端框架

| 选项 | 注解 | 适配本项目 |
|---|---|---|
| **Astro（推荐）** | 内容站/作品集首选；岛屿架构，默认不发 JS，仅交互组件"开岛"。`.astro` 是 HTML 超集，**现有原型可几乎原样粘贴**；最新 6.x 引入 Rust 版 Markdown 处理器，构建快；支持 API Endpoints + Vercel/Node 适配器承载动态接口；内置 i18n 路由。 | ★★★★★ 最贴合"UI 1:1 + 少量动态"，且零 JS 利于国内加载与备案 |
| Next.js | 全栈 React 框架（App Router、Server Components），生态最大、动态能力最强；但纯展示页会强制带 40–50KB 运行时，且把原型 HTML 改写成 JSX 有 UI 走样风险。 | ★★★☆ 适合重交互应用，本项目偏重，收益有限 |
| 纯静态 HTML + 独立函数 | 原型直接上线，动态接口用独立 Serverless 函数。改动最小，但缺模板化/组件复用/i18n，后续中英双语与多项目维护成本高。 | ★★★ 可作极简兜底，不利长期演进 |

> 结论：选 **Astro**。它在"保真原型"和"具备服务端能力"之间取得最佳平衡——展示页把原型 HTML/CSS 原样迁入 `.astro`，RAG 对话与后台等交互以 island 或服务端接口实现。备选 Next.js（若你更偏好 React 生态）。

### 2.2 数据库

| 选项 | 注解（2026 免费额度，以官网为准） | 适配 |
|---|---|---|
| **Supabase（推荐）** | 基于 Postgres 的一站式后端：DB + Auth + Storage + Edge Functions。免费版约 500MB 数据库、5 万月活认证用户、1GB 存储、50 万次边缘函数调用、2 个项目；不 scale-to-zero（约一周无活动后暂停）。**内置 Auth 直接用于后台登录**，省去自建。 | ★★★★★ 一站式，覆盖访问记录、RAG 记录、后台鉴权 |
| Neon | 纯 Serverless Postgres，scale-to-zero（约 500ms 唤醒）、分支（branching）强；免费版约 100 CU-hours/月、0.5GB/项目、可建 100 项目。但不含 Auth/Storage，需另配鉴权。 | ★★★★ 若只要数据库、想要 scale-to-zero |

> 结论：选 **Supabase**。它把数据库与后台登录一并解决，免费额度对个人站绰绰有余。若你只想要一个极简数据库、且自己解决登录，Neon 是优秀备选。

### 2.3 大语言模型（RAG 生成）

| 选项 | 注解（调研于 2026.06） | 适配 |
|---|---|---|
| **智谱 GLM-4.7-Flash（推荐）** | 2026 年初发布的 30B-A3B MoE 免费模型，200K 上下文、最长 128K 输出，编程/推理/Agent 表现为同尺寸开源 SOTA，国内直连、**可免费调用**。 | ★★★★★ 低流量个人站，免费即满足，且把"全量语料注入"完全装得下 |
| DeepSeek V4-Flash / V4-Pro | V4 系列支持百万级上下文；缓存命中价被打到极低（Flash 缓存命中约 0.02 元/百万 tokens），Pro 降价后缓存未命中输入约 3 元、输出约 6 元/百万 tokens；旧名 `deepseek-chat/reasoner` 将于 2026-07-24 弃用，对应 V4-Flash 非思考/思考模式。RAG 这类高缓存命中场景成本极低。 | ★★★★ 付费但极便宜，质量稳，作为收费备选/升级 |
| 通义千问 Qwen | 中文与生态成熟，亦有低价档位，可作第二备选。 | ★★★★ |

> 结论：主用 **智谱 GLM-4.7-Flash（免费）**；预留 DeepSeek V4-Flash 作为可切换的付费备选。**模型走服务端统一适配层**（OpenAI 兼容协议），切换只改配置，不动前端。

### 2.4 RAG / 向量化

语料规模很小（5–8 个项目 + 简历 + 经历），按需求文档判断**初期无需向量库**。

| 阶段 | 方案 | 注解 |
|---|---|---|
| **一期（推荐）** | **全量语料注入上下文** | 把结构化项目数据 + 简历整体作为系统上下文交给 GLM-4.7-Flash（200K 上下文足够）。零检索成本、零额外组件、回答天然带全局视野，便于"带引用"。 |
| 二期（语料膨胀后） | **pgvector + 硅基流动 bge-m3 嵌入** | Supabase 自带 pgvector；嵌入用硅基流动 bge-m3（免费、国内直连、中英文俱佳）或智谱 embedding-3（有免费额度）；纯中文追求精度可用 Qwen3-Embedding。 |

> 结论：一期直接全量注入，省去向量工程；把"是否上向量"留作语料增长后的演进开关。

### 2.5 邮件通知

| 选项 | 注解 | 适配 |
|---|---|---|
| **Resend（推荐）** | 事务邮件 API，免费额度足以覆盖个人站来访提醒（月级约数千封量级）；配合自有域名 + DNS（SPF/DKIM）发信不进垃圾箱，API 简洁。 | ★★★★★ |
| QQ/163 SMTP | 因提醒只发给你自己，最简方案；但 QQ 邮箱对第三方发信有会员/限制类坑。 | ★★★ 自收信够用，正式感弱 |
| 腾讯云 / 阿里云 邮件推送 | 国内合规、稳定，适合备案后国内阶段。 | ★★★★ 国内备选 |

> 结论：用 **Resend**；国内阶段如需可平滑切换到腾讯云邮件推送。提醒可设"即时 + 每日汇总"两档，避免打扰。

### 2.6 部署与 CDN / 备案

| 阶段 | 方案 | 注解 |
|---|---|---|
| **一期 · 即时上线** | **Vercel**（绑定海外/国际域名）+ Cloudflare 或腾讯云 EdgeOne "优选 IP/CNAME" 加速 | 当天上线、零备案、自动 HTTPS。Vercel 默认 CDN 未对大陆专门优化，国内访问偶慢/不稳，用优选 IP/反代可显著改善且无需备案。 |
| **二期 · 国内稳定** | **腾讯云 EdgeOne Pages**（"国内版 Vercel"，3200+ 节点含国内 2300+、免费 SSL、边缘 Serverless）或国内轻量云 + **ICP 备案** | 国内访问快而稳，对国内 HR 体验最佳。**ICP 备案约 1–3 周**，是国内上线唯一硬性时间瓶颈，建议注册域名当天并行启动。 |

> 结论：先 Vercel 跑通，再并行走 EdgeOne/备案。两阶段共用同一套代码，仅切换部署目标与域名解析。

### 2.7 鉴权

- **门禁（01-gate）**：非安全边界，定位"了解来访"。一次性表单 + Cookie/Session 维持（如 7 天免重复填），**防呆不防黑**。
- **后台（05-admin）**：用 **Supabase Auth**（仅本人账号），或服务端校验的强口令。后台 API 全部服务端鉴权，凭据绝不出现在前端。

## 3. 系统模块实现方案（对照原型 1:1）

> 通则：所有页面的 HTML 结构、class、配色变量（`--teal #0E8C82` / `--graphite #15191B` 等）、字阶、间距、动效，**严格沿用原型文件**。下文只描述"把静态原型接上数据与逻辑"的增量，不改视觉。

### 3.A 门禁页（01-gate.html）
- 原样迁移卡片、背景网格 SVG、输入校验（双必填才激活按钮）。
- 增量：提交时 `POST /api/visit` 写入访问记录并触发邮件；成功后跳转 `02-home`；写入会话标记实现 7 天免重复。
- 字段：公司名（必填）、职位（必填）、自动采集（时间、referer/UTM、设备、IP 粗粒度地域）。

### 3.B 主页（02-home.html）
- 原样迁移：Hero 大字 + 签名图形（城市网格→AI 节点）、能力画像、深色经历时间线、8 张项目卡、竞赛荣誉、技能、深色页脚、悬浮"问数字分身"、滚动渐入与 `prefers-reduced-motion` 降级。
- 增量：
  - 8 张项目卡内容**改为数据驱动**（从 `projects` 表或本地内容文件渲染），版式不变；
  - 卡片点击进入 03 详情；
  - **按访客公司轻定制**（呼应需求 5.4）：识别公司类型 → 调整项目卡排序；识别不到则默认排序，**绝不报错**；
  - 中英双语：Astro i18n，先中文、英文标 TODO。

### 3.C 项目详情（03-project-detail.html）
- 原样迁移弹层（dim 背景 + 卡片 + 概述/我的工作/产出/图集/外链）。
- 增量：做成**参数化模板**，按项目 id 渲染（8 个项目共用一套）；图集与外链来自数据；占位图后续替换。

### 3.D RAG 数字分身（04-rag-chat.html）
- 原样迁移侧滑面板、默认建议 chips、对话气泡、引用条、作品引用卡、输入框、免责说明。
- 增量：把原型里的演示 `demo{}` 替换为真实接口——
  - `POST /api/rag`：服务端组装"系统提示词 + 全量语料 + 历史"，调 GLM-4.7-Flash，返回**带引用**的回答与命中项目 id；
  - 引用卡点击跳转对应 03 详情；
  - 系统提示词约束：**仅基于语料、不编造、聚焦职业相关、与简历无关问题礼貌婉拒**；
  - 命中含图项目时回答下方展示缩略图。

### 3.E 后台看板（05-admin.html）
- 原样迁移指标卡、来访雷达、访问记录表、RAG 提问记录表。
- 增量：登录后从数据库拉真实数据；"来访雷达"按"重复来访/深度浏览"规则高亮；支持时间范围筛选；仅本人可见。

### 3.F 公共能力
- **埋点**：页面停留时长、浏览了哪些项目、RAG 提问，归并到当次访问会话。
- **i18n**：文案与语料中英两版（先中后英）。

## 4. 数据模型设计

```
visits（访问记录）
  id, company, role, visited_at, referrer, utm,
  device, region, duration_sec, viewed_projects(jsonb), session_id

rag_logs（RAG 提问记录）
  id, session_id, company, question, answer, cited_project_ids(jsonb), asked_at

projects（项目语料 / 展示）
  id, slug, title, role, period, tags(jsonb), summary,
  detail_overview, detail_work(jsonb), detail_outcome,
  images(jsonb), links(jsonb), sort_weight, lang

profile（简历/经历语料）
  id, section, content, lang   // 供 RAG 全量注入与页面展示复用
```

> `projects` 同时服务"页面展示"与"RAG 语料"，一份数据两处用，避免双维护（呼应需求文档 5.3）。

## 5. API 设计

| 接口 | 方法 | 鉴权 | 说明 |
|---|---|---|---|
| `/api/visit` | POST | 无（门禁） | 写访问记录 + 触发邮件提醒；返回会话标记 |
| `/api/track` | POST | 会话 | 上报停留时长/浏览项目（埋点） |
| `/api/rag` | POST | 会话 + **限流** | 代理 LLM，返回带引用回答；服务端持 key |
| `/api/admin/overview` | GET | **Supabase Auth** | 看板指标 + 访问记录 + RAG 记录 |
| `/api/mail`（内部） | — | 服务端 | 调 Resend 发提醒 |

> 所有对外接口均做输入长度/频率校验；`/api/rag` 额外做每会话/每 IP/每日封顶（见 §6）。

## 6. 安全与防护（重点：LLM API 防刷）

| 风险 | 对策 |
|---|---|
| LLM key 泄露 / 被刷量产生账单 | ① key **仅服务端**，前端永不接触；② `/api/rag` 服务端代理 + **限流**（每会话/每 IP 每分钟、每日上限）；③ **每日总调用封顶**，超限优雅降级（提示"今日问答已达上限"）；④ 单条提问 token 长度限制 |
| Prompt 注入 / 越权提问 | 系统提示词锁定范围；过滤注入指令；与职业无关问题婉拒 |
| 门禁被绕过 / 灌垃圾 | 基础校验 + 简单频控；接受"防呆不防黑" |
| 后台越权 | Supabase Auth 鉴权；后台 API 服务端校验；凭据不入前端 |
| 隐私合规 | 仅采集公司名/职位/粗粒度访问数据；门禁明示用途；数据仅本人可见 |

成本可控性：设单次问答成本 \(c\)、月问答次数 \(n\)，月费 \(C = c \times n\)。做了每日封顶后 \(n\) 有上限；又因主模型免费、备选模型缓存命中极低价，\(C\) 实际趋近于零。

## 7. 部署与备案方案

\[
\text{一期（当天）：Vercel + 优选IP} \;\longrightarrow\; \text{二期（并行 1–3 周）：EdgeOne/国内云 + ICP 备案}
\]

- **一期**：代码推 GitHub → Vercel 自动部署 → 绑定域名 → Cloudflare/EdgeOne 优选 IP 改善国内访问。零备案、自动 HTTPS。
- **二期**：域名注册当天启动 ICP 备案（瓶颈，1–3 周）；备案过后切到腾讯云 EdgeOne Pages 或国内轻量云，DNS 解析切换即可，代码不变。

| **Context7** | 注入版本化框架文档 | 防 Agent 用过时/幻觉 API；写 Astro/Supabase 代码时尤其重要 |
| **Supabase MCP**（远程 OAuth） | 直接建表、查数据、管 Auth/Edge Function | 数据模型、访问记录、RAG 记录全在 Supabase，让 Agent 直接操作 |
| **Playwright MCP**（或 Chrome DevTools MCP） | 浏览器自动化、截图、E2E | **守住"UI 1:1"的关键**——让 Agent 截图比对实现与原型，及时发现走样 |
| **GitHub MCP** | 仓库/PR/Issue 管理 | 代码托管与 Vercel 自动部署的联动 |
| **Figma MCP**（可选） | 暴露设计结构给 Agent | 你已有 Figma/Pixso，若把视觉细化稿放 Figma，可让 Agent 精确取值 |
| **Vercel MCP**（可选） | 部署/环境变量/日志 | 一期部署与排障 |

> 注意：Claude Code 已自带文件读写工具，**无需** Filesystem MCP；MCP 凭据放客户端密钥库/环境变量，不要硬编进配置；先用只读、需要时再开写操作。

### 9.2 推荐 Skills（Agent Skills，`.md` 形式）

2026 年的推荐范式是用 `.md` Skills 固化项目规则、组合多 MCP 工作流。建议为本项目建：

1. **`design-system.md`（最重要）**：把原型的设计 token（配色、字阶、圆角、间距、药丸按钮、动效规则）与"必须 1:1 还原原型、禁止自行发挥"写成硬约束，让每次生成都遵守，**从源头防 UI 走样**。
2. **`project-context.md`**：放本方案的架构、数据模型、API 约定、目录结构，作为 Agent 的长期上下文。
3. **`rag-prompt.md`**：沉淀数字分身的系统提示词（仅基于语料、带引用、婉拒无关问题），便于统一维护与迭代。
4. **`ui-verify.md`**：组合 Playwright MCP 的"实现 vs 原型截图比对"流程，作为每个页面的验收动作。

### 9.3 基础工具链
- 包管理与运行：Node LTS + pnpm；Astro CLI。
- 代码托管/部署：GitHub → Vercel（一期）。
- 环境变量：LLM key、Supabase key、Resend key **仅放服务端环境变量**，本地用 `.env`（不入库）。
- 质量：Lighthouse 自查（性能/可访问性）、移动端真机或模拟器验收。
