## 1. 项目背景与核心定位

### 1.1 背景

范诚昊为同济大学建筑与城市规划学院城乡规划专业本硕（2025.9 起硕士在读），近一年明显向 **AI 产品 / 城市数字化** 方向转型，已积累：

- 此芯科技 **AI 产品经理**实习（AI BOX / AI NAS 的市场研究、需求规划、方案表达）；
- 同济咨询咨询师助理（产业研究、可研编制、并搭建过可研 Work Flow 与项目谋划 Agent）；
- 个人 AI 项目（AI 热点监控推送工具、城市居民生活圈幸福度智能监测与推演模型）；
- 城市设计 / 调研报告国际竞赛获奖；
- 校足球队等社会活动。

### 1.2 核心定位（全站叙事主线）

> **"一个懂城市的 AI 产品经理"——AI PM 为主线，城乡规划专业背景作为不可复制的差异化壁垒。**

### 1.3 本项目的"元价值"（最重要的设计立意）

这个网站本身就是一件 **AI 产品作品**：它内置 RAG 数字分身、行为埋点看板、按访客定制内容等能力。

对一个 AI PM 候选人而言，HR 看到的不是"他声称会做 AI 产品"，而是**他已经独立做出来、正运行在眼前的一个 AI 产品**。因此：

- 站点的技术实现与产品细节，本身即为能力佐证，需在适当位置（如关于页 / 彩蛋）显性点出"本站由我用 AI 全流程搭建"；
- 一切设计取舍，优先服务于"让 HR 在 3 分钟内确信：这个人能把 AI 想法落地成可用产品"。

## 2. 目标用户与使用场景

### 2.1 主要用户：招聘方（HR / 用人部门负责人 / 猎头）

| 用户类型 | 关注点 | 行为特征 | 设计应对 |
|---|---|---|---|
| HR 初筛 | 是否匹配岗位、亮点是否清晰 | 停留短（1~3 分钟）、移动端为主 | 首屏极速传达定位；移动端体验优先 |
| 用人部门负责人 | 项目深度、真实能力、技术细节 | 停留较长、PC 端深读、会点开作品 | 项目详情 + RAG 深问 + 可外链 Demo/仓库 |
| 猎头 | 可匹配的岗位标签、亮点速记 | 快速扫描、可能转发 | 清晰标签化能力画像、易分享 |

### 2.2 次要用户：网站所有者本人（管理端）

- 查看"哪些公司、何时、看了什么、问了 RAG 什么"，反推求职意向命中情况；
- 维护语料库（项目、图片、外链）与文案。

### 2.3 典型场景

1. HR 拿到链接 → 移动端打开 → 填公司名 + 职位 → 进入 → 扫首屏定位 → 看 1~2 个 AI 项目 → 关闭。
2. 用人负责人 PC 打开 → 深读项目 → 用 RAG 问"他做过哪些落地的 AI 项目""他规划背景怎么帮到产品"→ 点开外链 Demo。
3. 范诚昊登录后台 → 看本周访问看板 → 发现某目标公司反复查看 → 主动跟进。

## 4. 模块 A：访问门禁层（硬门槛）

### 4.1 需求描述

进入个人展示页前，弹出登入界面，**必填**：

- A1 公司名称（必填，文本）
- A2 来访者职位（必填，文本 / 可给常见选项 + 自定义）

填写并提交后方可进入。设计目标：**门槛硬、体验轻**——不让人感觉被审问，而像"签到 / 报到"。

### 4.2 功能要点

| 编号 | 功能 | 说明 |
|---|---|---|
| A-1 | 强制门禁 | 未填写无法进入主站内容（含 RAG） |
| A-2 | 轻量体验 | 单屏、字段少、一句话说明"为方便我了解谁来访"，给予真诚感 |
| A-3 | 触发埋点 | 提交即记录：公司名、职位、时间、来源（referrer/UTM）、设备、地域（IP 粗粒度） |
| A-4 | 即时通知 | 提交即向本人发送邮件提醒（含公司名、职位、时间） |
| A-5 | 会话保持 | 同一访客一段时间内（如 7 天）免重复填写，避免反复打扰 |

### 4.3 亮点设计

- **不做账号体系**：HR 不会为看一份简历注册账号。仅用一次性表单 + 本地 Cookie/Session 维持，降低流失。
- **诚实化文案**：直接说明用途（"想知道是哪家公司在看我，方便我做针对性准备"），坦诚反而提升好感。
- **防呆而非防黑**：门禁目的是"了解来访者"，不是真正的安全隔离。因此不做强校验（不验证公司真伪），重点在埋点与体验，避免过度工程。

## 5. 模块 B：个人展示页（PC / 移动端自适应）

### 5.1 总体要求

- **响应式**：PC 端横向信息密度高、可并排；移动端单列、首屏高信息密度、触控友好。
- **首屏定位**：3 秒内让访客明白"懂城市的 AI 产品经理"。
- **叙事顺序**：定位 → 能力画像 → 代表项目 → 经历背书 → 联系。先给结论，再给证据。

### 5.2 区块明细

| 区块 | 内容 | PC 表现 | 移动端表现 |
|---|---|---|---|
| B1 Hero | 姓名、一句话定位、核心标签（AI 产品 / 城市数字化 / vibe coding）、CTA（看作品 / 问数字分身） | 大字 + 留白 + 右侧形象 | 单列居中、CTA 醒目 |
| B2 能力画像 | "AI PM × 城市规划"双引擎，3~4 条价值主张 | 卡片并排 | 纵向堆叠 |
| B3 经历时间线 | 实习（此芯、同济咨询）、教育（同济本硕） | 横向 / 竖向时间轴 | 竖向时间轴 |
| B4 项目作品集 | 5~8 个项目卡片 → 详情页/弹层 | 网格卡片 | 纵向卡片 |
| B5 竞赛荣誉 | WUPEN 竞赛、足球冠军、最佳营造奖 | 荣誉墙 | 列表 |
| B6 技能矩阵 | AI 应用 / 软件 / 语言 三类 | 分组标签 | 折叠分组 |
| B7 联系/简历 | 邮箱、电话、微信、简历 PDF 下载 | 页脚固定 | 底部 + 悬浮按钮 |

### 5.3 项目作品集（B4）数据结构

每个项目建议统一字段（同时服务页面展示与 RAG 语料）：

| 字段 | 说明 | 必填 |
|---|---|---|
| 标题 | 项目名 | 是 |
| 角色 | 我负责什么（PM / 数据 / 设计 / 全栈…） | 是 |
| 时间 | 起止 | 是 |
| 标签 | AI / 城市 / 竞赛 / 工具… 用于筛选与定制 | 是 |
| 一句话概述 | 卡片摘要 | 是 |
| 详细说明 | 背景 / 我的工作 / 产出 / 成果 | 是 |
| 图片组 | 效果图 / 截图 / 图纸 | 否 |
| 外链 | Demo / 仓库 / PPT / 报告 | 否 |

> 建议候选项目（待你确认最终入选，5~8 个）：① AI 热点监控推送工具 ② 生活圈幸福度智能监测推演模型 ③ 大都市乡村新产业新业态研究 ④ 川沙 1+1+5 乡村振兴连片研究 ⑤ WUPEN 城市设计竞赛二等奖 ⑥ WUPEN 调研报告竞赛 ⑦ 无为市商业街区业态策划 ⑧ 可研 Work Flow / 项目谋划 Agent 搭建。

### 5.4 亮点设计

- **按访客公司轻定制（呼应模块 D 数据）**：识别到 AI 类公司 → 首屏优先抬 AI 项目；规划院 / 设计院 → 优先抬规划项目。降级策略：识别不到则用默认排序，绝不报错。
- **"本站即作品"彩蛋**：在关于/页脚处点明本站由本人用 AI 全流程搭建，并可外链到本仓库或搭建过程说明。
- **中英双语切换**：覆盖外企 HR，一键切换；语料与文案需准备中英两版（可先中文，英文标 TODO）。

## 6. 模块 C：RAG 对话检索（升级为「AI 数字分身」）

### 6.1 需求描述

提供对话框，访客可用自然语言在"我的作品 / 成果 / 图片 / 经历"库中检索并获得**带引用的回答**，而非仅返回链接列表。页面默认给出搜索建议。

### 6.2 从"搜索"升级为"数字分身"的理由

- 单纯关键词搜索对一个 AI PM 候选人价值有限；
- 升级为"可对话的数字分身"后，HR 能直接问匹配性问题（如"他适合我们的 AI PM 岗位吗"），由模型基于真实语料作答——**这本身是最强的 AI 产品能力证明**。

### 6.3 功能要点

| 编号 | 功能 | 说明 |
|---|---|---|
| C-1 | 默认建议 | 进入即给 3~5 个可点建议，例：「他做过哪些 AI 项目？」「他的城市规划背景怎么帮到产品？」「自用 AI 工具库有哪些？」「他在此芯具体做了什么？」 |
| C-2 | 自然语言问答 | 基于语料库检索 + LLM 生成，**严禁脱离语料编造** |
| C-3 | 引用与跳转 | 回答标注来源项目，可一键跳到对应作品区块/详情 |
| C-4 | 富媒体结果 | 命中含图项目时，回答下方可展示对应图片缩略 |
| C-5 | 兜底话术 | 语料无相关内容时，诚实说明并引导联系本人，不硬编 |
| C-6 | 安全边界 | 仅回答与本人职业/项目相关问题；与简历无关问题礼貌婉拒，防 prompt 注入与刷量 |

### 6.4 语料与检索方案（需求层）

- **语料来源**：模块 B4 的结构化项目数据 + 简历正文 + 你补充的报告/图片说明。
- **检索方式**：项目数量不大（5~8 个 + 简历），**可不必上重型向量库**；初期用轻量方案（如把全部结构化语料作为上下文，或简单 embedding 检索）即可，避免过度工程。后续语料膨胀再升级向量检索。
- **模型**：国产 LLM API（DeepSeek / 通义 / 智谱），自带 key。
- **回答约束**：系统提示词限定"仅基于提供的语料、带引用、不编造、聚焦职业相关"。

### 6.5 风险提示（重要）

RAG 每次问答均调用付费 LLM API，必须做**用量防护**（详见 §9），否则存在被恶意刷量产生费用的风险。

## 7. 模块 D：管理后台（来访雷达 + RAG 记录）

### 7.1 需求描述

将"门禁收集的公司名/职位 + 行为数据"沉淀为一个轻量看板，让本人了解"谁在认真看我、关心哪段经历"。同时每次有效来访发邮件提醒。

### 7.2 功能要点

| 编号 | 功能 | 说明 |
|---|---|---|
| D-1 | 访问看板 | 列表 + 概览：公司名、职位、时间、停留时长、看了哪些项目、设备/地域 |
| D-2 | 来访雷达 | 高亮"反复来访 / 深度浏览"的公司，提示主动跟进 |
| D-3 | RAG 提问记录 | 谁问了什么 → 反推关注点（如频繁问 AI 项目，则该公司重 AI） |
| D-4 | 邮件提醒 | 每次有效来访即时邮件（公司名 + 职位 + 时间），可设每日汇总以防打扰 |
| D-5 | 语料维护 | 增删改项目、图片、外链、文案（初期可用配置文件/简单后台，不必复杂 CMS） |
| D-6 | 后台鉴权 | 仅本人可访问，简单密码/口令即可，注意不暴露在前端 |

### 7.3 亮点设计

- **意向雷达**：把"门槛"反向用成"销售线索分析"——你能反推哪些公司真在认真评估你。
- **隐私与诚实**：在门禁文案已告知用途，数据仅本人查看、不外传，符合最小必要原则。

## 8. 模块 E：上线与部署

### 8.1 策略：海外先行，国内备案并行（已确认）

| 阶段 | 方案 | 优点 | 代价 | 时间 |
|---|---|---|---|---|
| 第一阶段（即时上线） | 海外平台（Vercel / Netlify 等） | 当天上线、零备案、自动 HTTPS | 国内访问偶慢/不稳 | 1 天内 |
| 第二阶段（长期稳定） | 国内轻量云 + 国内域名 + ICP 备案 | 国内访问快而稳，对国内 HR 体验最佳 | 备案约 1~3 周、需资质 | 并行推进 |

### 8.2 域名与备案

- 域名：¥50/年级别（如 .com / .cn / .me）。建议尽早注册，备案以注册时间起算。
- 备案：若走国内服务器，需 ICP 备案（个人/主体资质、约 1~3 周）。**这是国内上线唯一的硬性时间瓶颈**，建议第一天就并行启动。
- 现状待确认：你目前**是否已有域名 / 服务器**（见 §13 待办）。

### 8.3 成本预算（量级，非精确报价）

| 项目 | 方案 | 预估 |
|---|---|---|
| 服务器 | 国产轻量云 / 海外免费额度 | ≤ ¥100/月 |
| 域名 | .com/.cn 等 | ~¥50/年 |
| LLM API | 国产、自带 key、按量 | 视用量，做了防护后可控（数元~数十元/月级） |
| 备案 | 自助 | ¥0（仅时间成本） |

> 说明：LLM 月费 ≈ 单次问答 token 成本 × 月问答次数。设单次成本 \(c\)、月问答次数 \(n\)，则月费 \(C = c \times n\)。做了 §9 的限流封顶后，\(n\) 有上限，费用即可控。

## 9. 安全与防护（重点：API 防刷）

| 风险 | 对策 |
|---|---|
| LLM API 被恶意刷量 → 账单暴涨 | ① API key **仅存服务端**，绝不出现在前端；② 服务端代理转发并**限流**（按 IP/会话每分钟、每日上限）；③ 设**每日总调用封顶**，超限优雅降级（提示"今日问答已达上限"）；④ 单条提问 token 长度限制 |
| Prompt 注入 / 越权提问 | 系统提示词限定回答范围；过滤明显注入指令；非职业相关问题婉拒 |
| 门禁被绕过 / 刷垃圾数据 | 基础校验 + 简单频控；接受"防呆不防黑"，门禁非安全边界 |
| 后台越权访问 | 后台独立鉴权，口令服务端校验，不在前端泄露 |
| 隐私合规 | 仅采集公司名/职位/粗粒度访问数据；门禁明示用途；数据仅本人可见 |
| 联系方式被爬 | 邮箱/电话做轻度防爬（如图片化或脚本渲染，权衡 SEO） |

## 11. 技术选型建议（需求层，非强制）

| 层 | 建议 | 理由 |
|---|---|---|
| 前端 | 现代前端框架 + 响应式 | 自适应、交互流畅、便于"作品感" |
| 后端 | 轻量服务端（承载 API 代理、埋点、邮件、后台） | 隐藏 LLM key、做限流、收数据 |
| 数据存储 | 轻量数据库 / 托管数据库 | 存访问记录、RAG 问答、语料 |
| LLM | 国产 API（DeepSeek/通义/智谱），自带 key | 便宜、国内快 |
| RAG | 初期轻量（全语料上下文或简单 embedding） | 语料少，避免过度工程 |
| 邮件 | 邮件服务（如 SMTP / 第三方邮件 API） | 来访提醒 |
